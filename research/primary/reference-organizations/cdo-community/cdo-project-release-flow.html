Project Release Flow

This page illustrates the release flow of CDO ontologies and consumer-projects of the ontologies. The “trunk” of the flow follows this path:

  • The CASE ontology repository tracks UCO’s ontology repository as a Git submodule, and creates releases with each release of UCO.
  • The CASE Python Utilities then compile a monolithic build of the CASE ontology, exporting that build via PyPI at case-utils.
  • A GitHub validation action, case-validation-action references the new case-utils version as a pip dependency, and uses that new version for validation with GitHub Actions in projects that adopt case-validation-action.

The flow shows various practices with consuming the UCO and CASE ontologies, frequently as part of data validation in Continuous Integration workflows.
